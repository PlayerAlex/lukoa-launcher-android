# Changelog

本文件只记录对外可感知、或会影响开发流程的重要变化。

## [0.9.1-beta9] - 2026-07-08

0.9.1-beta9 版本更新日志：
1. 修复实例管理里的高风险问题：如果旧配置里已经把主实例删没了，启动器现在会自动把实例配置扶正回来，不再容易卡在“只剩分身、恢复不回来”的状态。
2. 优化实例删除防呆：主实例现在默认保留，只有切换到对应分身实例后，才允许删除那个分身，减少误删后难恢复的情况。
3. 补充实例管理提示文案，让“当前为什么不能删、应该先切到哪里再删”这类信息更直白，少走弯路。

## [0.9.1-beta8] - 2026-07-08

0.9.1-beta8 版本更新日志：
1. 设置页现在可以按实例管理酒馆目录和端口，支持切换当前实例、新建分身实例，以及分别修改当前实例的路径和端口，给多实例使用打好了入口。
2. 优化自定义端口下的启动和识别链路：启动器会按当前实例端口去打开网页，首次启动成功识别和端口冲突识别也不再只盯着 `8000`。
3. 修复多实例下容易互相串状态的问题：Termux 的运行状态文件、PID、日志和回退记录现在会按实例分开保存，减少切换实例后读错状态、停错进程或串日志的风险。

## [0.9.1-beta7] - 2026-07-07

0.9.1-beta7 版本更新日志：
1. 优化启动前体检里的端口占用提示：如果酒馆端口已被别的进程占用，会优先提醒先处理端口，不再跳到后台权限等后续操作，避免用户误以为可以直接启动。
2. 优化启动酒馆前的预检说明：遇到端口占用时，会明确建议先关闭占用端口的应用，或重启 Termux/手机后重新体检。
3. 补充 Termux 状态推断测试，覆盖“正在启动”“前台会话退出”“已有进程但网页暂时打不开”等边界，减少重复启动或误判停止的风险。

## [0.9.1-beta6] - 2026-07-07

0.9.1-beta6 版本更新日志：
1. 进一步加固 Termux 唤醒链路：只有系统真正打开 Termux 后，才会记录唤醒冷却，避免唤醒失败后短时间内再点按钮却被误提示“刚刚已经触发过”。
2. 优化启动酒馆时的异常提示：启动命令已发出但系统没有自动打开 Termux 时，会明确提醒手动打开 Termux 查看日志，并提示先不要重复点启动。
3. 补充启动状态摘要测试，确保这种“命令已发送但需要手动打开 Termux”的场景仍被视为启动中，减少用户重复启动造成双进程的风险。

## [0.9.1-beta5] - 2026-07-07

0.9.1-beta5 版本更新日志：
1. 优化“唤醒 Termux 并返回”按钮，冷却中、正在唤醒、未找到 Termux 等情况会给出更准确的提示，不再只显示笼统的成功或失败。
2. 优化从后台切回启动器的链路：当 Termux 已安装、RUN_COMMAND 可用、但 Termux 后台常驻还没放行时，启动器会按冷却规则自动唤醒 Termux 并返回，减少后台被系统收掉后用户还以为酒馆能继续启动的情况。
3. 优化第一次启动酒馆前的非 iQOO 引导，按钮改为“唤醒 Termux 并返回”，会先打开 Termux 再回启动器，回来后再继续启动更顺。
4. 优化 iQOO 首次启动引导，Termux 后台常驻已经放行时不再重复弹出授权提醒。
5. 优化首次启动成功历史识别，已经检测到酒馆运行或网页可访问时，不再反复弹出首次启动引导。
6. 补充权限中心和体检里的 Termux 后台常驻说明，让首次启动酒馆、长任务、自动备份和前台日志同步的风险提示更清楚。

## [0.9.1-beta4] - 2026-07-03

0.9.1-beta4 版本更新日志：
1. 新增首次启动酒馆前的新手引导：iQOO 会提醒先给 Termux 开后台，其他手机会提醒先把 Termux 挂到小窗或分屏，减少第一次启动就被系统打断的情况。
2. 优化设置页里的权限展示，改成轻提示和详情入口，真正的运行状态继续集中放在启动页，界面更清楚。
3. 优化权限中心和诊断日志，补充 Termux 后台常驻检测，长任务、前台日志和自动备份遇到系统省电限制时更容易看出问题点。
4. 修复设置页状态信息塞得太满、打乱整体美化和阅读节奏的问题。
5. 修复之前只检查启动器自己的后台权限、漏掉 Termux 后台常驻状态的问题。
6. 修复新手第一次启动酒馆时，部分机型更容易因为 Termux 退到后台被系统打断、却缺少明确提醒的问题。

## [0.9.1-beta3] - 2026-07-03

### Changed

- App 内的更新日志改成更接近正常应用的编号样式，重要内容和新增内容会更靠前显示。
- Termux 命令页会优先显示当前会话的新增输出，并折叠超长模型列表和扩展列表，实时看日志更清楚。
- 横向滑动补了额外的解锁兜底，长时间使用后更不容易突然滑不动。

### Fixed

- 修复 GitHub 更新说明直接显示 `###`、`-` 这类 Markdown 标记，看起来像原始草稿的问题。
- 修复 Termux 日志同步容易夹带 ANSI 颜色码、旧会话内容和重复大段输出，导致实时查看困难的问题。
- 修复部分分区横滑偶发被卡住，滑一段时间后突然无法继续左右切页的问题。

## [0.9.1-beta2] - 2026-07-03

### Changed

- 更新和回退现在都会先弹出统一确认框，先把当前版本、目标版本和当前源说清楚，再执行危险操作。
- 首次安装向导补充了默认版本、默认路径和安装后的下一步提示，新手更容易沿着一条线装到能启动。
- 恢复酒馆备份成功后，会明确提示旧目录保留位置和后续该做的检查，不再让人猜恢复完还要不要补动作。

### Fixed

- 修复版本管理里更新、回退仍沿用双击确认的交互不一致问题。
- 修复恢复完成后提示过短、容易误以为已经自动处理完依赖和善后步骤的问题。

## [0.9.1-beta1] - 2026-07-03

### Changed

- 应用备份前会先显示备份名、时间、大小和恢复目标目录，恢复前更容易确认自己将覆盖什么。
- 版本管理进一步收口，当前 Git 源、官方版本列表和更新/回退按钮状态会一起判断，减少错源和误点。
- 诊断日志补充了更多权限状态和一键体检摘要，反馈问题时不用再手抄那么多信息。

### Fixed

- 修复 Termux 实时同步时旧日志可能反复堆回界面的问题，App 里会优先显示新增内容。
- 修复版本列表和当前 Git 源不一致时仍可能继续更新或回退的风险提示缺口。

## [0.9.0] - 2026-07-03
