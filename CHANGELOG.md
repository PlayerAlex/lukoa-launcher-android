# Changelog

## [0.9.1-beta24] - 2026-07-10

0.9.1-beta24 版本更新日志：
1. 打开备份恢复确认时，备份名称、时间和大小会改在后台读取，减少读取外部备份信息时界面卡顿或无响应的情况。
2. 读取备份信息期间会显示明确提示并阻止重复操作；如果备份无法读取，不会进入恢复确认流程。
3. 当前选中的酒馆实例正在运行或启动时，版本管理会直接阻止更新和回退，并提示先停止当前实例，减少误操作风险。

本文件只记录对外可感知、或会影响开发流程的重要变化。

## [0.9.1-beta23] - 2026-07-10

0.9.1-beta23 版本更新日志：
1. 优化安装、更新、回退、备份和恢复等长任务的结果等待方式，减少任务执行期间持续卡顿和掉帧的情况。
2. Termux 返回结果现在会在后台完成保存和后续整理，命令完成后的反馈更稳定，也减少了返回瞬间界面无响应的风险。
3. 收紧诊断历史的体积并去除重复输出，在保留冷启动恢复和问题排查能力的同时，降低长期使用后的存储与解析负担。

## [0.9.1-beta22] - 2026-07-10

0.9.1-beta22 版本更新日志：
1. 修复历史日志较大时，导出运行日志或诊断日志可能长时间卡住、界面无响应的问题。现在会直接读取最近的有效日志，不再先把整份历史文件加载进内存。
2. 超大日志被截断时会在导出文件里明确说明，并保留最近的日志内容，既方便继续排查，也能减少内存占用和闪退风险。

## [0.9.1-beta21] - 2026-07-10

0.9.1-beta21 版本更新日志：
1. 启动页移除了“Termux 前台回传”窗口，日志区域现在只保留真正需要查看的酒馆运行日志，页面更直接。
2. Termux 底层回传仍会用于状态判断、诊断和导出，不会因为窗口移除影响启动、检测或问题排查功能。

## [0.9.1-beta20] - 2026-07-09

0.9.1-beta20 版本更新日志：
1. 启动页里的 Termux 区块继续收口，显示内容会更接近你在真正 Termux 前台看到的当前启动会话，而不是把整段酒馆运行日志混在一起顶上来。
2. 运行日志导出链路改到后台处理，日志较大时不该再因为生成导出文件把界面直接卡死。
3. 导出的日志内容也重新分段整理了，Termux 前台回传、酒馆运行日志和 App 操作反馈会更容易分开看，排查问题时少一点对不上号的困惑。

## [0.9.1-beta19] - 2026-07-09

0.9.1-beta19 版本更新日志：
1. “强制释放端口 / 清理残留进程”入口已挪到设置页的“诊断与日志”里，不再和启动、停止按钮混在一起，排查问题时更容易按用途去找。
2. Termux 调用返回里不再显示 `==== SillyTavern live log ... ==== ` 和 `==== end SillyTavern live log ==== ` 这类包裹标记，界面会直接保留里面真正的日志内容。
3. 如果当前没检测到需要强制清理的残留进程，入口提示也会更直接，少一点“点了没反应”的困惑。

## [0.9.1-beta18] - 2026-07-08

0.9.1-beta18 版本更新日志：
1. Termux 调用返回现在尽量按原样显示，不再额外加时间戳，也不再拆成“实时新增”“新增日志”“最新重要日志”这类二次标签，排查问题时更容易直接对照 Termux 里的内容。
2. 普通命令返回和后台日志同步现在统一按同一种展示方式处理，减少同一段输出被重复整理、截断或换一种说法后看不明白的情况。
3. “强制释放端口 / 清理残留进程”入口现在会固定显示，不再只在特定状态下偷偷出现，想手动处理残留进程时更容易找到。

## [0.9.1-beta17] - 2026-07-08

0.9.1-beta17 版本更新日志：
1. 停止酒馆现在默认只做温和停止，不会再在普通停止里悄悄升级成强制清理，减少多实例场景下误杀残留进程的风险。
2. 新增独立的“强制释放端口 / 强制清理残留进程”入口。只有普通停止没退干净，或明确检测到当前实例残留进程、端口占用时，才会提示你手动确认继续。
3. 强制清理前的确认信息补得更清楚了。现在会把当前实例、当前目录、当前端口，以及为什么建议强制清理写出来，方便你先确认不会清到别的实例。
4. 启动前预检、状态摘要和相关提示也一起收口，普通停止、残留进程和端口占用的区别会更直白，少一点“到底该点哪个按钮”的困惑。

## [0.9.1-beta16] - 2026-07-08

0.9.1-beta16 版本更新日志：
1. 修复多实例下长任务容易串结果的问题。安装、更新、回退、备份、恢复这类操作，现在会优先按当前实例识别回传结果，减少切到别的分身后把旧任务结果认到当前实例上的情况。
2. 修复端口占用时的状态误判。现在如果酒馆端口其实是被别的进程占用，启动页、状态摘要和一键体检会直接提示端口冲突，不再误显示成“酒馆正在运行”或错误引导你去点“停止酒馆”。
3. 优化路径防呆。`~/`、`$HOME/`、带 `.` / `..` 或重复斜杠的目录现在会更稳地按同一个路径识别，减少分身实例误碰主实例保留目录或目录同址识别漏判的问题。
4. 修复“选择检测到的目录”会误带未保存端口输入的问题。现在切到检测到的目录时，会保留当前实例已经保存好的端口，不会顺手把临时输入框里的端口一起带过去。

## [0.9.1-beta15] - 2026-07-08

0.9.1-beta15 版本更新日志：
1. 启动前的预检弹窗现在会直接显示当前实例、当前目录和当前端口，点启动前就能先肉眼确认这次准备启动的是哪一个实例，减少误启动到别的目录的担心。
2. 这次显示的是当前启动器已经选中的实例配置，不会因为 `start.sh` 文件名相同就自动跑去主实例，方便你在处理权限、路径或环境问题时顺手一起确认目标实例有没有选对。

## [0.9.1-beta14] - 2026-07-08

0.9.1-beta14 版本更新日志：
1. 修复分身实例误把主实例托管目录当成候选目录、迁移目标甚至启动目录的问题。`~/LukoaLauncher/SillyTavern` 现在会一直保留给主实例，分身实例默认继续从 `~/LukoaLauncher/SillyTavern2` 往后分配。
2. 补上启动前防呆：如果当前实例路径误指到主实例保留目录，启动器会先拦住并提示去改路径，避免分身实例启动到主实例的酒馆。
3. 修复一类“备份明显变慢、长时间不结束”的问题：如果酒馆把 `dataRoot` 配成了上级大目录，备份现在会直接阻止并说明原因，不再傻等很久。
4. 优化手动备份超时后的兜底处理：如果备份文件其实已经生成，只是 Termux 最终返回丢了，启动器会自动补识别并把备份补回列表，减少白等十分钟后还以为失败的情况。
5. 补强路径同址识别：`~/...`、`$HOME/...` 和 Termux 绝对路径现在会按同一个目录处理，减少迁移后按钮状态不对、默认目录识别失准和实例路径占用漏判的问题。

## [0.9.1-beta13] - 2026-07-08

0.9.1-beta13 版本更新日志：
1. 修复实例迁移后的路径识别串状态问题：只有“目录不存在”确实还指向当前实例当前路径时，才会再弹候选目录；旧实例或旧路径留下的结果，不该再把你刚迁好的实例重新拉去选目录。
2. 优化路径设置里的实例刷新。切换实例、迁移目录成功，或删除分身实例后，路径设置会按当前实例重新刷新路径类型和迁移按钮状态，减少还要切去别的实例再切回来的情况。
3. 修复迁移到新目录后仍可能被误判成“酒馆还在运行”或“还在启动”的问题，收紧进程识别条件，减少旧目录残留或无关进程导致的假运行状态。

## [0.9.1-beta12] - 2026-07-08

0.9.1-beta12 版本更新日志：
1. 酒馆默认安装目录改成启动器托管位置。主实例默认会用 `~/LukoaLauncher/SillyTavern`，新建分身实例也会自动分配各自的独立目录，减少和旧目录、反代目录混在一起的风险。
2. 设置页新增酒馆目录迁移入口，支持把当前实例迁移到启动器托管目录、主实例的传统默认目录 `~/SillyTavern`，或你自己填写的自定义地址；迁移前会先拦住运行中实例、重复目录和实例冲突目录。

4. 手动备份和自动备份的默认下载目录统一改到 `Download/LukoaLauncher/backups/sd` 和 `Download/LukoaLauncher/backups/zd`，旧的 `Download/lukoa/backups` 不再继续兼容，也不再回退到 Termux 私有备份目录。

## [0.9.1-beta6] - 2026-07-07

0.9.1-beta5 版本更新日志：

4. 优化 iQOO 首次启动引导，Termux 后台常驻已经放行时不再重复弹出授权提醒。
5. 优化首次启动成功历史识别，已经检测到酒馆运行或网页可访问时，不再反复弹出首次启动引导。
6. 补充权限中心和体检里的 Termux 后台常驻说明，让首次启动酒馆、长任务、自动备份和前台日志同步的风险提示更清楚。

## [0.9.0] - 2026-07-03

### Changed

- 新增仓库级 `AGENTS.md` 和项目技能，后续 AI 进入仓库时会先看到发版文案规则和模块化开发规则。
- 发布脚本不再默默生成泛泛而谈的默认公告；发版时必须明确提供正式公告或有意识地使用自动公告。

### Fixed

- 修复发布流程里“先改版本、先提交、最后才发现没写公告”的风险，改成在真正动版本号前先检查公告输入。

## [0.8.33-beta9] - 2026-07-03

### Changed

- 将 Android Lint 纳入 CI，推送和提 PR 时会自动跑静态检查并上传报告。
- 继续补充纯逻辑模块的单元测试，覆盖包源解析、体检报告解析、备份命令编码、Termux 日志增量提取等回归高发点。
- 补齐项目维护基础文件，包括编辑器规范、换行规范、贡献说明、安全说明、Issue/PR 模板和依赖更新配置。

### Fixed

- 修复 `LukoaLauncherScreen` 中已弃用的 `LocalLifecycleOwner` 引用，避免后续升级 Compose/Lifecycle 时继续积累维护警告。
- 补上最基础的 release 构建配置，避免项目只验证 debug、不验证 release 的维护盲区。

## [0.8.33-beta8] - 2026-07-03

### Changed

- 将启动器主界面的大型组合文件拆成导航、启动、版本管理、设置、对话框等模块，降低后续修改风险。
- 为仓库补上 Gradle Wrapper、GitHub Actions CI、基础测试入口和发布脚本防呆。

### Fixed

- 修复拆分界面后跨文件共用组件的可见性问题，恢复项目可编译状态。
- 修复发布脚本在 GitHub Release 不存在时容易中断的问题。
- 修复发布脚本默认把未跟踪草稿文件一起提交的风险。

## [0.8.31] - 2026-07-02

### Fixed

- 修复自动备份设置重启后回到默认值的问题。
- 修复停止酒馆不稳定的问题，改成弹窗确认后再停止。
- 修复镜像源和版本列表错源、缓存错用的问题。
- 修复备份恢复提示里会误导用户的安全备份文案。

### Changed

- 酒馆路径不再由脚本自动接管，只提供候选目录供 App 侧选择。
- 安装预检改成按本次实际使用的仓库去读取和检测。
