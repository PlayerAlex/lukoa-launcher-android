# Changelog

本文件只记录对外可感知、或会影响开发流程的重要变化。

## [0.9.1-beta1] - 2026-07-03

### Changed

- 应用备份前会先显示备份名、时间、大小和恢复目标目录，恢复前更容易确认自己将覆盖什么。
- 版本管理进一步收口，当前 Git 源、官方版本列表和更新/回退按钮状态会一起判断，减少错源和误点。
- 诊断日志补充了更多权限状态和一键体检摘要，反馈问题时不用再手抄那么多信息。

### Fixed

- 修复 Termux 实时同步时旧日志可能反复堆回界面的问题，App 里会优先显示新增内容。
- 修复版本列表和当前 Git 源不一致时仍可能继续更新或回退的风险提示缺口。

## [0.9.0] - 2026-07-03

### Changed

- 新增仓库级 `AGENTS.md` 和项目技能，后续 AI 进入仓库时会先看到发版文案规则和模块化开发规则。
- 发布脚本不再默默生成泛泛而谈的默认公告；发版时必须明确提供正式公告或有意识地使用自动公告。

### Fixed

- 修复发布流程里“先改版本、先提交、最后才发现没写公告”的风险，改成在真正动版本号前先检查公告输入。

## [0.8.33-beta9] - 2026-07-03

### Changed

- 将 Android Lint 纳入 CI，推送和提 PR 时会自动跑静态检查并上传报告。
- 继续补充纯逻辑模块的单元测试，覆盖包源解析、体检报告解析、备份命令编码、Termux 日志增量提取等回归高发点。
- 补齐项目维护基础文件，包括编辑器规范、换行规范、贡献说明、安全说明、Issue/PR 模板和依赖更新配置。

### Fixed

- 修复 `LukoaLauncherScreen` 中已弃用的 `LocalLifecycleOwner` 引用，避免后续升级 Compose/Lifecycle 时继续积累维护警告。
- 补上最基础的 release 构建配置，避免项目只验证 debug、不验证 release 的维护盲区。

## [0.8.33-beta8] - 2026-07-03

### Changed

- 将启动器主界面的大型组合文件拆成导航、启动、版本管理、设置、对话框等模块，降低后续修改风险。
- 为仓库补上 Gradle Wrapper、GitHub Actions CI、基础测试入口和发布脚本防呆。

### Fixed

- 修复拆分界面后跨文件共用组件的可见性问题，恢复项目可编译状态。
- 修复发布脚本在 GitHub Release 不存在时容易中断的问题。
- 修复发布脚本默认把未跟踪草稿文件一起提交的风险。

## [0.8.31] - 2026-07-02

### Fixed

- 修复自动备份设置重启后回到默认值的问题。
- 修复停止酒馆不稳定的问题，改成弹窗确认后再停止。
- 修复镜像源和版本列表错源、缓存错用的问题。
- 修复备份恢复提示里会误导用户的安全备份文案。

### Changed

- 酒馆路径不再由脚本自动接管，只提供候选目录供 App 侧选择。
- 安装预检改成按本次实际使用的仓库去读取和检测。
