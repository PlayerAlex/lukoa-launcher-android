# Changelog

## [0.9.2-beta3] - 2026-07-10

0.9.2-beta3 版本更新日志：

1. 修复酒馆已经进入启动流程、但网页还没准备好时，主按钮错误变回“启动酒馆”或停止入口消失的问题。
2. 启动过程中主按钮会保持为红色“停止酒馆”，用户无需等待启动超时即可主动停止当前实例。
3. 实时日志状态判断只读取启动器生成的顶部状态信息，不再让聊天内容或运行日志中的 JSON 字段误改酒馆的启动、运行或停止状态。
4. 停止确认入口现在同时支持“运行中”和“启动中”，减少重复启动以及无法及时中止启动流程的情况。
## [0.9.2-beta2] - 2026-07-10

0.9.2-beta2 版本更新日志：

1. 修复酒馆已经停止、端口也没有监听时，实时日志同步仍可能把残留的 Termux 命令进程误判成酒馆正在运行的问题。
2. 加强酒馆进程身份校验，现在会同时核对进程工作目录、实际可执行程序和精确启动参数，不再只凭命令文本里出现 `server.js` 或 `start.sh` 判断。
3. 实时日志只负责同步日志和确认停止状态，不再仅凭候选进程把已停止的实例重新标记为运行中。
4. 清理失效或已经被其他进程复用的旧 PID 记录，减少停止、迁移、更新和回退被错误拦截的情况。

## [0.9.2-beta1] - 2026-07-10

0.9.2-beta1 版本更新日志：

1. 清理已经停用且没有调用入口的旧界面流程，减少后续功能更新时误接回旧安装、权限、状态或备份交互的风险。
2. 同步项目模块导航与当前代码结构，后续修复会更容易定位到正确功能区域，降低改动波及其他页面的可能。
3. 清理开发过程中留下的临时补丁、发布草稿和本地依赖目录，并增加忽略规则，避免这些文件再次混入版本改动。
4. 当前启动、安装、权限、版本管理、备份恢复和多实例操作方式保持不变。

## [0.9.1] - 2026-07-10

0.9.1 正式版汇总了 beta1 至 beta31 的测试与优化：

1. 新增并完善多实例管理，可为不同实例分别设置酒馆目录和端口，运行状态、PID、日志、版本与回退记录按实例隔离，减少切换实例后串状态或误操作。
2. 完善酒馆目录识别与迁移，支持传统目录、启动器托管目录和自定义目录；迁移、删除和选择候选目录时会检查运行状态、路径占用与实例冲突，并保留清楚的危险操作确认。
3. 加固备份与恢复流程，增加恢复预览、备份名称与路径校验、后台读取和导出、大文件防卡顿、超时归档补识别，以及自动备份与其他高风险操作之间的互斥保护。
4. 统一安装、更新和回退确认，明确当前版本、目标版本和 Git 源；切换源后会重新读取对应版本列表，减少错源更新、错误回退和版本信息混用。
5. 优化首次启动、Termux 唤醒、后台常驻和权限引导，针对命令已发送但 Termux 未打开、后台被系统限制等情况给出更准确的下一步提示，减少重复启动。
6. 区分温和停止与强制清理。普通停止不再自动升级为强制清理；端口占用或残留进程需要用户从诊断入口明确确认，降低多实例下误杀进程的风险。
7. 改善酒馆运行日志与诊断体验，日志保留 ANSI 颜色并减少重复、截断和无关包裹内容；日志和诊断导出改为后台处理，大日志下更不容易卡住界面。
8. 长时间安装、更新、回退、备份、恢复和目录迁移使用持久化操作锁、独立命令标记和未完成任务恢复，App 重建后仍能继续检查，减少旧结果串入新任务或按钮提前解锁。
9. 优化端口占用、路径同址、候选目录、运行状态和当前实例识别，减少把其他进程误判为酒馆、把旧实例结果套到当前实例或把临时端口输入误保存的问题。
10. 对话框、备份、实例路径和镜像设置的状态进一步隔离，现有按钮、确认方式、默认 `~/SillyTavern` 和用户自定义路径保持不变。

0.9.1-beta14 版本更新日志：
