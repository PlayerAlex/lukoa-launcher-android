# 文档导航

这个目录专门放仓库里的长期文档，尽量把“面对用户的说明”和“面对开发/AI 协作的说明”分开放。

## 目录

### `design/`

给 UI / UX 重构使用：

- `UI_REDESIGN_BRIEF.md`
  新信息架构、页面状态、组件清单、设计交付和工程承接边界

### `project-management/`

给项目长期推进和 AI 交接使用：

- `README.md`
  项目管理资料的说明页
- `AI新对话启动模板.md`
  新开 AI 对话时可直接复制的启动模板
- `lukua-launcher-AI项目管理模板.xlsx`
  版本总表、发版检查、兼容性测试、AI 协作记录

### `development/`

给继续维护仓库结构和代码入口的人使用：

- `repository-map.md`
  仓库结构、关键文件入口和建议阅读顺序

## 维护原则

- 面向普通用户的说明优先留在根目录 `README.md`
- 面向开发、发版、交接、AI 协作的资料尽量收进 `docs/`
- 新增文档前先判断它是“用户文档”还是“开发文档”，不要混放
