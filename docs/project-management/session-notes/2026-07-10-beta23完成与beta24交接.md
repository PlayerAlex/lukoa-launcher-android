## 2. beta23 已完成

beta23 的唯一核心目标是减轻 Termux 结果等待和接收链路的主线程负担，已经完成：

1. 新增事件驱动的 `TermuxResultRepository`，实时结果通过共享流通知等待者。
2. `TavernController` 不再每 500ms 读取 SharedPreferences 并解析整段历史 JSON。
3. 保留 executionId 优先、命令名与开始时间兼容兜底、nonce 匹配逻辑。
4. SharedPreferences 只负责冷启动恢复、最新完整结果和有限诊断历史。
5. 新增 `TermuxCommandTimeoutPolicy`，集中管理安装、更新、回退、备份、恢复等超时。
6. Receiver 使用 `goAsync()`；Receiver 和 Service 的解析、持久化均在 IO 后台执行。
7. 自动备份完成后的目录扫描、排序和删除改为一次性 WorkManager 任务。
8. 历史结果每条文本最多 16000 字符；`raw` 不再重复保存 stdout、stderr 和已知字段。
9. 已补通知、executionId、命令时间兜底、nonce、超时、冷启动、历史上限和 raw 去重测试。

不要在 beta24 重新实现这些内容，不要恢复轮询。

## 5. 备份恢复预览后台化

当前证据：

- `BackupRestorePreviewResolver.resolve()` 会调用 `BackupLibraryFiles.describeLibraryArchive()`，可能执行 MediaStore 查询或文件信息读取。
- `LukoaLauncherScreen.openApplyBackupPreview()` 当前在点击事件路径同步调用 resolver，再打开恢复确认框。
- 这条链路仍可能在主线程读取较慢的外部备份信息。

关键文件：

- `app/src/main/java/moe/lukoa/launcher/BackupRestorePreview.kt`
- `app/src/main/java/moe/lukoa/launcher/BackupLibraryFiles.kt`
- `app/src/main/java/moe/lukoa/launcher/LukoaLauncherScreen.kt`
- `app/src/main/java/moe/lukoa/launcher/LauncherDialogs.kt`

推荐实现：

1. 在现有 Compose 协程作用域中用 `withContext(Dispatchers.IO)` 读取预览。
2. 增加“正在读取备份信息”状态，读取期间阻止重复点击。
3. 用请求 token、路径比对或取消旧 Job，避免旧路径结果覆盖新选择。
4. 读取失败时给出清楚提示，不进入恢复确认，也不能直接开始恢复。
5. 保留路径校验、PendingLauncherTask、安全备份、恢复确认和 Shell 最终校验。
6. 请求有效性或状态归并优先拆成纯 Kotlin 逻辑并补测试。

至少补测试：

- 新请求覆盖旧请求。
- 旧路径预览不会应用到新选择。
- 读取失败不会进入确认流程。
- 时间和大小格式化测试；可顺手消除当前默认 Locale 的 Lint 警告，但不要改变显示含义。

## 6. 更新和回退运行状态防呆

当前证据：

- `TavernVersionActionGuards` 已检查安装状态、当前版本、本地改动、目标版本和 Git 源。
- `requestTavernVersionAction()` 已检查全局忙碌状态和 PendingLauncherTask。
- 当前没有在 App 侧明确拦截 `tavernRunning` 或 `tavernStarting`，仍可能先打开确认框，再依赖 Shell 拒绝。

关键文件：

- `app/src/main/java/moe/lukoa/launcher/TavernVersionActionGuards.kt`
- `app/src/main/java/moe/lukoa/launcher/LauncherVersionManagementSection.kt`
- `app/src/main/java/moe/lukoa/launcher/LukoaLauncherScreen.kt`
- `app/src/main/java/moe/lukoa/launcher/TavernVersionActionConfirmation.kt`

推荐实现：

1. 把“当前实例正在运行或启动中”纳入纯 Kotlin guard。
2. 在按钮禁用状态和点击入口同时阻止，提示“先停止当前实例，再更新或回退”。
3. 只检查当前选中 profile，不能误判其他实例。
4. 保留 Shell 的最终安全校验和更新、回退前的自动安全备份。

至少补测试：

- 正在运行时更新和回退不可用。
- 正在启动时更新和回退不可用。
- 停止后原有版本关系、Git 源和本地改动规则仍正常。
- 禁用原因文案清楚一致。
